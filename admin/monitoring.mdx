CBDB Online integrates Prometheus metrics collection for monitoring application performance, request patterns, and system health.

## Metrics Endpoint

Prometheus metrics are exposed at the `/metrics` endpoint, implemented in `app/Http/Controllers/MetricsController.php`.

<Card title="Metrics Endpoint" icon="gauge">
  **URL:** `https://your-domain.com/metrics`
  
  **Format:** Prometheus text-based exposition format
  
  **Authentication:** Optional (configurable)
</Card>

## Available Metrics

The system collects three primary metrics:

### HTTP Request Total

<ParamField query="cbdb_http_requests_total" type="Counter">
  Total number of HTTP requests received
  
  **Labels:**
  - `method` - HTTP method (GET, POST, PUT, DELETE, etc.)
  - `path` - Route pattern (e.g., `/basicinformation/{personid}/edit`)
  - `status` - HTTP status code (200, 404, 500, etc.)
  
  **Example:**
  ```prometheus
  cbdb_http_requests_total{method="GET",path="/basicinformation/{personid}/edit",status="200"} 1523
  cbdb_http_requests_total{method="POST",path="/api/person/create",status="201"} 87
  ```
</ParamField>

### HTTP Request Duration

<ParamField query="cbdb_http_request_duration_seconds" type="Histogram">
  Distribution of HTTP request processing times
  
  **Labels:**
  - `method` - HTTP method
  - `path` - Route pattern
  - `status` - HTTP status code
  
  **Buckets:** 0.005, 0.01, 0.025, 0.05, 0.1, 0.25, 0.5, 1.0, 2.5, 5.0, 10.0 seconds
  
  **Example:**
  ```prometheus
  cbdb_http_request_duration_seconds_bucket{method="GET",path="/",status="200",le="0.1"} 1450
  cbdb_http_request_duration_seconds_bucket{method="GET",path="/",status="200",le="0.25"} 1498
  cbdb_http_request_duration_seconds_sum{method="GET",path="/",status="200"} 156.78
  cbdb_http_request_duration_seconds_count{method="GET",path="/",status="200"} 1523
  ```
</ParamField>

### HTTP Requests In Progress

<ParamField query="cbdb_http_requests_in_progress" type="Gauge">
  Number of HTTP requests currently being processed
  
  **Labels:**
  - `method` - HTTP method
  
  **Note:** Path is **intentionally omitted** to avoid label mismatch issues during route resolution.
  
  **Example:**
  ```prometheus
  cbdb_http_requests_in_progress{method="GET"} 3
  cbdb_http_requests_in_progress{method="POST"} 1
  ```
</ParamField>

<Note>
  **Why no `path` label on in_progress metric?**
  
  The gauge is incremented before route resolution (when `$request->route()` is null) and decremented after. Using path would cause label mismatch between increment/decrement operations, leading to incorrect gauge values.
</Note>

## Configuration

### Environment Variables

Configure Prometheus metrics in `.env`:

```env
# Enable/disable metrics collection
PROMETHEUS_ENABLED=true

# Namespace for metrics (prefix)
PROMETHEUS_NAMESPACE=cbdb

# Storage adapter (memory/redis/apc)
PROMETHEUS_STORAGE_ADAPTER=memory

# Enable HTTP metrics middleware
PROMETHEUS_HTTP_METRICS_ENABLED=true

# ⚠️ CRITICAL: Prevent memory leak
PROMETHEUS_INCLUDE_ROUTE_PARAMS=true

# IP whitelist (comma-separated, empty = allow all)
PROMETHEUS_ALLOWED_IPS=

# Basic authentication (optional)
PROMETHEUS_AUTH_ENABLED=false
PROMETHEUS_AUTH_USERNAME=prometheus
PROMETHEUS_AUTH_PASSWORD=secret
```

### Storage Adapters

<Tabs>
  <Tab title="Memory (Default)">
    **Best for:** Development, single-process applications
    
    ```env
    PROMETHEUS_STORAGE_ADAPTER=memory
    ```
    
    **Pros:**
    - No external dependencies
    - Fast and simple
    
    **Cons:**
    - Not shared across processes
    - Resets on application restart
  </Tab>
  
  <Tab title="Redis (Recommended for Production)">
    **Best for:** Production, multi-process, distributed deployments
    
    ```env
    PROMETHEUS_STORAGE_ADAPTER=redis
    PROMETHEUS_REDIS_HOST=127.0.0.1
    PROMETHEUS_REDIS_PORT=6379
    PROMETHEUS_REDIS_PASSWORD=
    PROMETHEUS_REDIS_DB=2
    ```
    
    **Pros:**
    - Shared across all processes and servers
    - Persists across application restarts
    - Supports distributed architecture
    
    **Cons:**
    - Requires Redis server
    - Slight performance overhead (1-2ms per request)
  </Tab>
  
  <Tab title="APC">
    **Best for:** PHP-FPM multi-process environments
    
    ```env
    PROMETHEUS_STORAGE_ADAPTER=apc
    ```
    
    **Pros:**
    - Fast, in-memory storage
    - Shared across PHP-FPM workers
    
    **Cons:**
    - Requires APC PHP extension
    - Not shared across multiple servers
  </Tab>
</Tabs>

## Access Control

### IP Whitelisting

Restrict metrics access to specific IP addresses or networks:

```env
# Single IP
PROMETHEUS_ALLOWED_IPS=10.0.0.1

# Multiple IPs
PROMETHEUS_ALLOWED_IPS=10.0.0.1,10.0.0.2

# CIDR notation (IPv4)
PROMETHEUS_ALLOWED_IPS=192.168.1.0/24

# CIDR notation (IPv6)
PROMETHEUS_ALLOWED_IPS=2001:db8::/32

# Mixed formats
PROMETHEUS_ALLOWED_IPS=10.0.0.1,192.168.1.0/24,2001:db8::/32
```

**Implementation:** `MetricsController.php:43-71`

The IP whitelist check supports:
- Exact IP matching
- CIDR notation for IPv4 and IPv6
- Multiple entries (comma-separated)
- Empty list = allow all IPs

### HTTP Basic Authentication

Add password protection to the metrics endpoint:

```env
PROMETHEUS_AUTH_ENABLED=true
PROMETHEUS_AUTH_USERNAME=prometheus
PROMETHEUS_AUTH_PASSWORD=your_secure_password
```

**Usage:**

```bash
# With authentication
curl -u prometheus:your_secure_password https://your-domain.com/metrics

# Using Prometheus scrape config
scrape_configs:
  - job_name: 'cbdb-laravel'
    basic_auth:
      username: 'prometheus'
      password: 'your_secure_password'
    static_configs:
      - targets: ['your-domain.com:443']
```

**Implementation:** `MetricsController.php:165-179`

## Critical: Preventing Memory Leaks

<Warning>
  **MUST SET:** `PROMETHEUS_INCLUDE_ROUTE_PARAMS=true`
  
  Failure to set this will cause severe memory leaks!
</Warning>

### The Problem

When `include_route_params = false`:

- ❌ Each unique URL creates a **separate metric time series**
- ❌ Example: `/basicinformation/1/edit`, `/basicinformation/2/edit`, `/basicinformation/3/edit`
- ❌ 10,000 person records = 10,000+ time series
- ❌ Each time series includes Counter + Histogram (14 buckets) + Gauge
- ❌ **Memory continuously grows until OOM**

### The Solution

When `include_route_params = true` (default):

- ✅ All requests to the same route share **one time series**
- ✅ Example: All `/basicinformation/{personid}/edit` requests merged
- ✅ Fixed memory footprint
- ✅ Metrics aggregated by route pattern (more useful)

### Verification

Check your metrics output:

**❌ Wrong (memory leak):**
```prometheus
cbdb_http_requests_total{method="GET",path="/basicinformation/1/edit",status="200"} 1
cbdb_http_requests_total{method="GET",path="/basicinformation/2/edit",status="200"} 1
cbdb_http_requests_total{method="GET",path="/basicinformation/3/edit",status="200"} 1
...(thousands more lines)
```

**✅ Correct (memory safe):**
```prometheus
cbdb_http_requests_total{method="GET",path="/basicinformation/{personid}/edit",status="200"} 3
```

### Memory Impact Calculation

For a site with 10,000 person records:

**Wrong configuration:**
```
10,000 URLs × (1 Counter + 14 Histogram buckets + 1 Gauge) = 160,000 objects ≈ 100+ MB
```

**Correct configuration:**
```
50 routes × (1 Counter + 14 Histogram buckets + 1 Gauge) = 800 objects ≈ 5 MB
```

## Prometheus Server Configuration

Add CBDB Online as a scrape target:

```yaml prometheus.yml
scrape_configs:
  - job_name: 'cbdb-laravel'
    scrape_interval: 15s
    scrape_timeout: 10s
    metrics_path: '/metrics'
    scheme: https
    
    static_configs:
      - targets:
        - 'input.cbdb.fas.harvard.edu'
    
    # If using basic auth
    basic_auth:
      username: 'prometheus'
      password: 'your_secure_password'
    
    # If using IP whitelist, ensure Prometheus server IP is allowed
```

## Useful Queries

### Request Rate (QPS)

```promql
# Overall request rate
rate(cbdb_http_requests_total[5m])

# Request rate by path
sum by (path) (rate(cbdb_http_requests_total[5m]))

# Request rate by status code
sum by (status) (rate(cbdb_http_requests_total[5m]))
```

### Error Rate

```promql
# 5xx error rate
sum(rate(cbdb_http_requests_total{status=~"5.."}[5m])) 
  / 
sum(rate(cbdb_http_requests_total[5m]))

# 4xx error rate
sum(rate(cbdb_http_requests_total{status=~"4.."}[5m])) 
  / 
sum(rate(cbdb_http_requests_total[5m]))
```

### Response Time (Latency)

```promql
# Average latency
rate(cbdb_http_request_duration_seconds_sum[5m]) 
  / 
rate(cbdb_http_request_duration_seconds_count[5m])

# P50 latency
histogram_quantile(0.50, rate(cbdb_http_request_duration_seconds_bucket[5m]))

# P95 latency
histogram_quantile(0.95, rate(cbdb_http_request_duration_seconds_bucket[5m]))

# P99 latency
histogram_quantile(0.99, rate(cbdb_http_request_duration_seconds_bucket[5m]))
```

### Concurrent Requests

```promql
# Current concurrent requests
sum(cbdb_http_requests_in_progress)

# By method
sum by (method) (cbdb_http_requests_in_progress)
```

### Top Slowest Endpoints

```promql
# Average duration by path (descending)
topk(10, 
  rate(cbdb_http_request_duration_seconds_sum[5m]) 
    / 
  rate(cbdb_http_request_duration_seconds_count[5m])
)
```

## Troubleshooting

### Metrics Endpoint Returns 403

**Cause:** IP whitelist blocking your request

**Solution:**
```env
# Option 1: Add your IP to whitelist
PROMETHEUS_ALLOWED_IPS=your.ip.address.here

# Option 2: Disable whitelist (development only)
PROMETHEUS_ALLOWED_IPS=
```

### Metrics Endpoint Returns 401

**Cause:** Basic authentication enabled but credentials not provided

**Solution:**
```bash
# Provide credentials
curl -u username:password https://your-domain.com/metrics

# Or disable authentication (development only)
PROMETHEUS_AUTH_ENABLED=false
```

### Memory Usage Growing

**Cause:** `include_route_params = false` causing memory leak

**Solution:**
```env
# Enable route parameter grouping
PROMETHEUS_INCLUDE_ROUTE_PARAMS=true
```

**Verify fix:**
```bash
# Check metric output
curl https://your-domain.com/metrics | grep "basicinformation"

# Should see: path="/basicinformation/{personid}/edit"
# Not: path="/basicinformation/123/edit"
```

**Clear existing metrics:**
```bash
# If using Memory adapter
php artisan optimize:clear

# If using Redis adapter
redis-cli KEYS "PROMETHEUS_*" | xargs redis-cli DEL
```

### Metrics Not Collecting

**Check configuration:**
```bash
# Verify metrics enabled
grep PROMETHEUS_ENABLED .env

# Verify HTTP metrics enabled
grep PROMETHEUS_HTTP_METRICS_ENABLED .env
```

**Check middleware registration:**
```bash
# Should see PrometheusMetrics middleware
php artisan route:list
```

## Performance Impact

Metrics collection overhead:

| Storage Adapter | Per-Request Overhead | Notes |
|----------------|---------------------|-------|
| Memory | < 0.1ms | Negligible impact |
| Redis | 1-2ms | Acceptable for most applications |
| APC | < 0.5ms | Low overhead |

<Tip>
  If metrics collection impacts performance, consider:
  
  - Using memory adapter in development
  - Sampling only a percentage of requests
  - Disabling metrics entirely with `PROMETHEUS_ENABLED=false`
</Tip>
