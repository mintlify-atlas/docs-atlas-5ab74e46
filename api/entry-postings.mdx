## Query People by Entry Path

Query people who entered service through specific entry paths, with support for place and time period filtering.

### Endpoint

```bash
POST /api/query_entry_postings
```

<Note>
This endpoint uses **POST** method with `Content-Type: application/json`.
</Note>

### Request Body Parameters

<ParamField body="entry" type="array" required>
  Array of entry path IDs to query
</ParamField>

<ParamField body="usePeoplePlace" type="number" default={0}>
  Enable person-related place filtering:
  - `1`: Enable
  - `0`: Disable (ignores `peoplePlace` and `locationType`)
</ParamField>

<ParamField body="peoplePlace" type="array">
  Array of person-related place IDs (only used if `usePeoplePlace=1`)
</ParamField>

<ParamField body="locationType" type="string">
  Type of place to filter (only used if `usePeoplePlace=1`):
  - `pAddr`: Only person's place
  - `eAddr`: Only entry place
  - `peAddr`: Both person's place and entry place
</ParamField>

<ParamField body="useDate" type="number" default={0}>
  Enable date filtering:
  - `1`: Enable
  - `0`: Disable (ignores all date parameters)
</ParamField>

<ParamField body="dateType" type="string">
  Date type (only used if `useDate=1`):
  - `entry`: Entry year
  - `index`: Index year
  - `dynasty`: Dynasty
</ParamField>

<ParamField body="dateStartTime" type="number">
  Start year (only used if `dateType=entry` or `dateType=index`)
</ParamField>

<ParamField body="dateEndTime" type="number">
  End year (only used if `dateType=entry` or `dateType=index`)
</ParamField>

<ParamField body="dynStart" type="number">
  Dynasty start code (only used if `dateType=dynasty`)
</ParamField>

<ParamField body="dynEnd" type="number">
  Dynasty end code (only used if `dateType=dynasty`)
</ParamField>

<ParamField body="useXy" type="number" default={0}>
  Include coordinate data:
  - `1`: Include
  - `0`: Exclude
</ParamField>

<ParamField body="start" type="number" default={1}>
  Starting record number (1-based)
</ParamField>

<ParamField body="list" type="number" default={10}>
  Number of records to return
</ParamField>

### Example Request

<CodeGroup>
```bash Query by entry year
curl -X POST "https://input.cbdb.fas.harvard.edu/api/query_entry_postings" \
  -H "Content-Type: application/json" \
  -d '{
    "entry": [36],
    "usePeoplePlace": 0,
    "peoplePlace": [],
    "locationType": "peAddr",
    "useDate": 1,
    "dateType": "entry",
    "dateStartTime": 1368,
    "dateEndTime": 1644,
    "dynStart": null,
    "dynEnd": null,
    "useXy": 1,
    "start": 1,
    "list": 10
  }'
```

```bash Query by dynasty
curl -X POST "https://input.cbdb.fas.harvard.edu/api/query_entry_postings" \
  -H "Content-Type: application/json" \
  -d '{
    "entry": [36],
    "usePeoplePlace": 0,
    "peoplePlace": [],
    "locationType": "peAddr",
    "useDate": 1,
    "dateType": "dynasty",
    "dateStartTime": null,
    "dateEndTime": null,
    "dynStart": 17,
    "dynEnd": 20,
    "useXy": 1,
    "start": 1,
    "list": 10
  }'
```
</CodeGroup>

<Note>
The first example queries people who became jinshi (進士) between 1368-1644.  
The second example queries people who became jinshi during Jin Dynasty to Qing Dynasty (dynasties 17-20).
</Note>

<Warning>
**Important Notes:**

- The `use*` parameters have higher priority than their corresponding data arrays
- When `dateType=entry` or `dateType=index`, only `dateStartTime` and `dateEndTime` are used
- When `dateType=dynasty`, only `dynStart` and `dynEnd` are used (other date fields are ignored)
</Warning>
