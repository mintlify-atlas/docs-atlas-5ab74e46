## Query People by Place

Query people associated with specific places through various relationship types (individual, entry, office posting, etc.).

### Endpoint

```bash
POST /api/query_place
```

<Note>
This endpoint uses **POST** method with `Content-Type: application/json`.
</Note>

### Request Body Parameters

<ParamField body="peoplePlace" type="array" required>
  Array of place IDs to query
</ParamField>

<ParamField body="placeType" type="array" required>
  Array of place relationship types to include:
  - `individual`: Person's place
  - `entry`: Entry path place
  - `association`: Social association place
  - `officePosting`: Office posting place
  - `institutional`: Institutional affiliation place
  - `kinship`: Kinship-related place
  - `associate`: Associate's place
</ParamField>

<ParamField body="useDate" type="number" default={0}>
  Enable date filtering:
  - `1`: Enable
  - `0`: Disable (ignores all date parameters)
</ParamField>

<ParamField body="dateType" type="string">
  Date type (only used if `useDate=1`):
  - `index`: Index year
  - `dynasty`: Dynasty
</ParamField>

<ParamField body="dateStartTime" type="number">
  Start year (only used if `dateType=index`)
</ParamField>

<ParamField body="dateEndTime" type="number">
  End year (only used if `dateType=index`)
</ParamField>

<ParamField body="dynStart" type="number">
  Dynasty start code (only used if `dateType=dynasty`)
</ParamField>

<ParamField body="dynEnd" type="number">
  Dynasty end code (only used if `dateType=dynasty`)
</ParamField>

<ParamField body="useXy" type="number" default={0}>
  Include coordinate data:
  - `1`: Include
  - `0`: Exclude
</ParamField>

<ParamField body="start" type="number" default={1}>
  Starting record number (1-based)
</ParamField>

<ParamField body="list" type="number" default={10}>
  Number of records to return
</ParamField>

### Example Request

<CodeGroup>
```bash Query by index year
curl -X POST "https://input.cbdb.fas.harvard.edu/api/query_place" \
  -H "Content-Type: application/json" \
  -d '{
    "peoplePlace": [2928, 10522, 12553, 13947, 13949],
    "placeType": ["individual", "entry", "officePosting"],
    "useDate": 1,
    "dateType": "index",
    "dateStartTime": 1200,
    "dateEndTime": 1644,
    "dynStart": null,
    "dynEnd": null,
    "useXy": 1,
    "start": 1,
    "list": 10
  }'
```

```bash Query by dynasty
curl -X POST "https://input.cbdb.fas.harvard.edu/api/query_place" \
  -H "Content-Type: application/json" \
  -d '{
    "peoplePlace": [2928, 10522, 12553, 13947, 13949],
    "placeType": ["individual", "entry", "officePosting"],
    "useDate": 1,
    "dateType": "dynasty",
    "dateStartTime": null,
    "dateEndTime": null,
    "dynStart": 17,
    "dynEnd": 22,
    "useXy": 1,
    "start": 1,
    "list": 10
  }'
```
</CodeGroup>

<Note>
The first example queries people from specified places (individual, entry, or office posting) during 1200-1644 CE.  
The second example queries people from specified places during Jin Dynasty to Qing Dynasty (dynasties 17-22).
</Note>

<Warning>
**Important Notes:**

- `useDate` parameter has higher priority - when set to `0`, all date-related parameters are ignored
- When `dateType=index`, only `dateStartTime` and `dateEndTime` are used
- When `dateType=dynasty`, only `dynStart` and `dynEnd` are used (other date fields are ignored)
- Multiple place types can be specified to get comprehensive results about a person's connections to places
</Warning>

<Note>
**Use Case:**

This endpoint is ideal for:
- Finding all people associated with specific geographic locations
- Analyzing population distribution across different place types
- Regional historical research with flexible time period filtering
</Note>
