## Query Office Postings

Query office appointment records (除授記錄) with support for office, place, and time period filtering.

### Endpoint

```bash
POST /api/query_office_postings
```

<Note>
This endpoint uses **POST** method with `Content-Type: application/json`.
</Note>

### Request Body Parameters

<ParamField body="office" type="array" required>
  Array of office IDs to query
</ParamField>

<ParamField body="useOfficePlace" type="number" default={0}>
  Enable office-related place filtering:
  - `1`: Enable
  - `0`: Disable (ignores `officePlace`)
</ParamField>

<ParamField body="officePlace" type="array">
  Array of office-related place IDs (only used if `useOfficePlace=1`)
</ParamField>

<ParamField body="usePeoplePlace" type="number" default={0}>
  Enable person-related place filtering:
  - `1`: Enable
  - `0`: Disable (ignores `peoplePlace`)
</ParamField>

<ParamField body="peoplePlace" type="array">
  Array of person-related place IDs (only used if `usePeoplePlace=1`)
</ParamField>

<ParamField body="useDate" type="number" default={0}>
  Enable date filtering:
  - `1`: Enable
  - `0`: Disable (ignores all date parameters)
</ParamField>

<ParamField body="dateType" type="string">
  Date type (only used if `useDate=1`):
  - `index`: Index year
  - `dynasty`: Dynasty
</ParamField>

<ParamField body="indexStartTime" type="number">
  Index year start (only used if `dateType=index`)
</ParamField>

<ParamField body="indexEndTime" type="number">
  Index year end (only used if `dateType=index`)
</ParamField>

<ParamField body="dynStart" type="number">
  Dynasty start code (only used if `dateType=dynasty`)
</ParamField>

<ParamField body="dynEnd" type="number">
  Dynasty end code (only used if `dateType=dynasty`, inclusive)
</ParamField>

<ParamField body="useXy" type="number" default={0}>
  Include coordinate data:
  - `1`: Include
  - `0`: Exclude
</ParamField>

<ParamField body="start" type="number" default={0}>
  Starting record number (0-based)
</ParamField>

<ParamField body="list" type="number" default={65535}>
  Number of records to return
</ParamField>

### Example Request

<CodeGroup>
```bash Full example with all filters
curl -X POST "https://input.cbdb.fas.harvard.edu/api/query_office_postings" \
  -H "Content-Type: application/json" \
  -d '{
    "office": [920, 1022, 1023],
    "useOfficePlace": 0,
    "officePlace": [],
    "usePeoplePlace": 1,
    "peoplePlace": [2928, 10522, 12553, 13947, 13949],
    "useDate": 1,
    "dateType": "index",
    "indexStartTime": 960,
    "indexEndTime": 1250,
    "dynStart": null,
    "dynEnd": null,
    "useXy": 1,
    "start": 11,
    "list": 10
  }'
```

```bash Simple office query
curl -X POST "https://input.cbdb.fas.harvard.edu/api/query_office_postings" \
  -H "Content-Type: application/json" \
  -d '{
    "office": [950],
    "useOfficePlace": 0,
    "officePlace": [],
    "usePeoplePlace": 0,
    "peoplePlace": [],
    "useDate": 0,
    "dateType": "index",
    "indexStartTime": null,
    "dynStart": null,
    "dynEnd": null,
    "useXy": 0,
    "start": 0,
    "list": 65535
  }'
```
</CodeGroup>

<Note>
The first example queries appointment records for Prime Minister positions (宰相/左丞相/右丞相) held by people from specific places (興化/興化軍) during Song Dynasty (960-1250), returning records 11-20.

The second example retrieves all records for office ID 950 (知州) in Song Dynasty without place restrictions.
</Note>

<Warning>
**Important Notes:**

- `office` parameter is **required** - office ID array cannot be empty
- To query all appointments in a region without office filtering, use the [Query by Place](/api/query-by-place) endpoint instead
- The `use*` parameters have higher priority than their corresponding data arrays
- When `dateType=dynasty`, the query includes records from `dynStart` to `dynEnd` (inclusive)
</Warning>
