## Query Relatives

Query kinship relationships for given people with customizable relationship distance parameters.

### Endpoint

```bash
POST /api/query_relatives_2
```

<Note>
This endpoint uses **POST** method with `Content-Type: application/json`.
</Note>

### Request Body Parameters

<ParamField body="people" type="array" required>
  Array of person IDs to query
</ParamField>

<ParamField body="mCircle" type="number" default={0}>
  Use five-degree mourning circle (五服) query:
  - `0`: Use five-degree mourning circle (ignores custom distance parameters)
  - `1`: Use custom distance parameters (below)
</ParamField>

<ParamField body="MAncGen" type="number">
  Maximum ancestor generation distance (only used if `mCircle=1`)
</ParamField>

<ParamField body="MDecGen" type="number">
  Maximum descendant generation distance (only used if `mCircle=1`)
</ParamField>

<ParamField body="MColLink" type="number">
  Maximum collateral (sibling) relationship distance (only used if `mCircle=1`)
</ParamField>

<ParamField body="MMarLink" type="number">
  Maximum marriage relationship distance (only used if `mCircle=1`)
</ParamField>

<ParamField body="MLoop" type="number">
  Maximum loop iterations (only used if `mCircle=1`)
</ParamField>

### Example Request

```bash
curl -X POST "https://input.cbdb.fas.harvard.edu/api/query_relatives_2" \
  -H "Content-Type: application/json" \
  -d '{
    "people": [1762],
    "mCircle": 0,
    "MAncGen": 1,
    "MDecGen": 1,
    "MColLink": 1,
    "MMarLink": 1,
    "MLoop": 2
  }'
```

<Note>
This example queries relatives of Wang Anshi (王安石, person ID 1762) using custom parameters:
- Maximum 1 generation upward
- Maximum 1 generation downward  
- Maximum 1 collateral relationship
- Maximum 1 marriage relationship
</Note>

<Warning>
**Important Notes:**

- `mCircle` parameter has higher priority than custom distance parameters (`MAncGen`, `MDecGen`, `MColLink`, `MMarLink`, `MLoop`)
- When `mCircle=0`, the query uses the five-degree mourning circle (五服) system, ignoring custom parameters
- When returning the root person record, `rKinship` is set to `'ego'`, and all `p*` fields return empty strings
</Warning>
