## What is CBDB Online?

CBDB Online (中國歷代人物傳在線記錄入系統) is a collaborative web-based biographical data entry system for the China Biographical Database Project (CBDB). It enables researchers, scholars, and volunteers worldwide to contribute biographical information about historical Chinese figures spanning multiple dynasties.

<Note>
The system is hosted at [https://input.cbdb.fas.harvard.edu/](https://input.cbdb.fas.harvard.edu/) and is maintained by the CBDB Project team.
</Note>

## Who Is It For?

### Researchers and Scholars
Academics studying Chinese history can directly contribute verified biographical data to the CBDB, ensuring their research findings are preserved and accessible to the global scholarly community.

### Crowdsourcing Contributors
Volunteers with knowledge of historical Chinese figures can participate in data entry, helping expand the database's coverage under expert supervision.

### Database Administrators
Project coordinators and data curators can manage user accounts, review proposals, maintain code tables, and monitor data quality through administrative interfaces.

### Developers
The system provides APIs for programmatic access to biographical data, supports MCP (Model Context Protocol) for read-only queries, and offers Prometheus metrics for monitoring.

## Technical Foundation

CBDB Online is built on modern web technologies:

- **Backend**: Laravel 12.x framework running on PHP 8.2+ (PHP 8.4 recommended)
- **Database**: MariaDB 10.3.39 with SQLite support for testing and exports
- **Frontend**: AdminLTE 3 (Bootstrap 4) with Vue 3 components, built using Vite
- **Web Server**: Caddy
- **Authentication**: Laravel Sanctum for API token management

## Data Scope

The database covers:

- **Biographical records** (BIOG_MAIN): Core person information including names, dynasties, life spans
- **Addresses** (ADDR_CODES, BIOG_ADDR_DATA): Geographic locations and addresses associated with individuals
- **Offices and Postings** (OFFICE_CODES, POSTING_DATA): Official positions and appointments
- **Kinship Relations** (KIN_DATA): Family relationships and genealogical connections
- **Social Associations** (ASSOC_DATA): Professional and social connections
- **Literary Works** (TEXT_CODES, TEXT_INSTANCE_DATA): Authored texts and publications
- **Code Tables**: Standardized reference data for dynasties, entry types, status codes, and more

<Tip>
All table schemas and relationships are documented in the [Database Schema Guide](/reference/database-schema) for developers who need to understand the underlying data structure.
</Tip>
