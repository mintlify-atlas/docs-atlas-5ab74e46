CBDB Online 提供自動化腳本，將生產環境的 MariaDB 數據庫匯出為 SQLite 格式，並同步到 HuggingFace 公開數據集供研究者下載使用。

## 概述

### 功能特點

<CardGroup cols={2}>
  <Card title="自動化匯出" icon="database">
    - 匯出 77 個 CBDB 數據表
    - 產生 SQLite 格式資料庫
    - 生成 SHA-256 校驗和 metadata
  </Card>
  
  <Card title="HuggingFace 同步" icon="cloud-upload">
    - 自動上傳到公開數據集
    - 歷史版本管理（按月歸檔）
    - 提供 latest.zip 最新版本
  </Card>
</CardGroup>

## 腳本架構

### 腳本位置

- **匯出腳本**: `scripts/export-daily-sqlite.sh` - 將 MySQL/MariaDB 表格匯出為 SQLite
- **同步腳本**: `scripts/weekly-sqlite-sync.sh` - 匯出、壓縮並上傳到 HuggingFace

### 工作流程

`weekly-sqlite-sync.sh` 執行以下步驟：

<Steps>
  <Step title="前置檢查">
    確認 `zip`、`hf` 已安裝且 HuggingFace 認證有效
  </Step>
  
  <Step title="匯出數據庫">
    呼叫 `export-daily-sqlite.sh` 產生：
    - `cbdb_YYYYMMDD.sqlite3` - SQLite 資料庫檔案
    - `cbdb_YYYYMMDD.json` - Metadata (SHA-256, 時間戳等)
  </Step>
  
  <Step title="壓縮檔案">
    產生 `cbdb_YYYYMMDD.zip`（zip 內使用平面檔名，不包含絕對路徑）
  </Step>
  
  <Step title="上傳到 HuggingFace">
    以單一 commit 上傳以下檔案：
    - `history/cbdb_YYYYMM/cbdb_YYYYMMDD.zip` - 歷史歸檔
    - `latest.zip` - 最新版本
    - `metadata/YYYY-MM/YYYY-MM-DD.json` - Metadata
    - `latest.json` - 最新版本 metadata
  </Step>
  
  <Step title="清理">
    刪除所有臨時檔案（包括原始 SQLite 匯出檔與 metadata）
  </Step>
</Steps>

## 匯出腳本詳解

### export-daily-sqlite.sh

此腳本負責從 MySQL/MariaDB 匯出 77 個 CBDB 數據表到 SQLite：

#### 腳本配置

```bash
# 輸出目錄（默認：db-data）
OUTPUT_DIR="${OUTPUT_DIR:-db-data}"

# 源數據庫連接名稱（默認：mysql）
SOURCE_DB="${SOURCE_DB:-mysql}"

# 日期後綴
DATE_SUFFIX=$(date +%Y%m%d)

# 輸出檔案
OUTPUT_FILE="${OUTPUT_DIR}/cbdb_${DATE_SUFFIX}.sqlite3"
OUTPUT_META_FILE="${OUTPUT_DIR}/cbdb_${DATE_SUFFIX}.json"
```

## 同步腳本詳解

### weekly-sqlite-sync.sh

此腳本負責壓縮匯出的 SQLite 並上傳到 HuggingFace：

#### 前置檢查

```bash
check_requirements() {
    # 檢查 zip 命令
    if ! command -v zip &> /dev/null; then
        echo "錯誤: 缺少 zip"
        exit 1
    fi
    
    # 檢查 hf CLI
    if ! command -v hf &> /dev/null; then
        echo "錯誤: 缺少 hf (請安裝: pipx install huggingface-hub)"
        exit 1
    fi
    
    # 檢查 HuggingFace 認證
    if ! hf auth whoami &> /dev/null; then
        echo "錯誤: 未找到 HuggingFace 認證"
        exit 1
    fi
}
```

#### 壓縮流程

```bash
# 壓縮為 zip（zip 內為平面檔名）
zip -j -9 "$ZIP_FILE" "$SQLITE_FILE" "$META_FILE"
```

<Note>
使用 `-j` 參數確保 zip 內只包含檔案名稱，不包含目錄路徑。使用 `-9` 參數進行最大壓縮。
</Note>

#### 上傳流程

上傳使用單一 commit 包含所有檔案：

```bash
# 準備暫存目錄
mkdir -p "${HF_STAGE_DIR}/history/${HF_MONTH}"
cp "$ZIP_FILE" "${HF_STAGE_DIR}/${HF_ZIP_PATH}"
cp "$ZIP_FILE" "${HF_STAGE_DIR}/latest.zip"

if [ -f "$META_FILE" ]; then
    mkdir -p "${HF_STAGE_DIR}/metadata/${META_MONTH}"
    cp "$META_FILE" "${HF_STAGE_DIR}/${META_PATH}"
    cp "$META_FILE" "${HF_STAGE_DIR}/latest.json"
fi

# 上傳整個目錄（所有檔案在同一個 commit）
hf upload "$HF_DATASET_REPO" \
    "$HF_STAGE_DIR" . \
    --repo-type dataset \
    --commit-message "Update CBDB SQLite (${META_DATE})"
```

#### 清理流程

```bash
cleanup() {
    # 刪除暫存目錄
    rm -rf "$HF_STAGE_DIR"
    
    # 刪除原始 SQLite 匯出檔
    rm -f "$SQLITE_FILE"
    
    # 刪除原始 zip
    rm -f "$ZIP_FILE"
    
    # 刪除 metadata
    rm -f "$META_FILE"
}
```

<Warning>
清理函數會在腳本退出時自動執行（使用 `trap cleanup EXIT`），確保不會留下臨時檔案。
</Warning>

## 執行方式

### 手動執行

<Tabs>
  <Tab title="完整同步流程">
    ```bash
    bash scripts/weekly-sqlite-sync.sh
    ```
    
    此命令會執行完整流程：匯出 → 壓縮 → 上傳 → 清理
  </Tab>
  
  <Tab title="僅匯出 SQLite（不上傳）">
    ```bash
    bash scripts/export-daily-sqlite.sh
    ```
    
    此命令僅匯出 SQLite，不進行壓縮和上傳。生成的檔案位於 `db-data/` 目錄。
  </Tab>
</Tabs>

### Cron 定時任務

設置每週日凌晨 3 點自動執行：

<Steps>
  <Step title="編輯 crontab">
    ```bash
    crontab -e
    ```
  </Step>
  
  <Step title="添加定時任務">
    ```bash
    # 每週日凌晨 3 點執行（GMT+8）
    0 3 * * 0 /path/to/cbdb-online-main-server/scripts/weekly-sqlite-sync.sh >> /var/log/cbdb-sqlite-sync.log 2>&1
    ```
  </Step>
  
  <Step title="驗證設置">
    ```bash
    # 查看 crontab 列表
    crontab -l
    ```
  </Step>
</Steps>

<Note>
Cron 執行時的環境變量可能與交互式 shell 不同。腳本已處理 `HOME` 和 `PATH` 環境變量，確保 `hf` 命令可用（位於 `~/.local/bin/`）。
</Note>

## 檔案組織結構

HuggingFace 數據集的檔案組織結構：

```
cbdb-sqlite/
├── latest.zip                          # 最新版本（符號連結）
├── latest.json                         # 最新版本 metadata
├── history/                            # 歷史歸檔
│   ├── cbdb_202412/                   # 2024年12月
│   │   ├── cbdb_20241201.zip
│   │   ├── cbdb_20241208.zip
│   │   └── cbdb_20241215.zip
│   └── cbdb_202501/                   # 2025年1月
│       └── cbdb_20250105.zip
└── metadata/                           # Metadata 歸檔
    ├── 2024-12/
    │   ├── 2024-12-01.json
    │   ├── 2024-12-08.json
    │   └── 2024-12-15.json
    └── 2025-01/
        └── 2025-01-05.json
```

### 命名規則

- **SQLite 檔案**: `cbdb_YYYYMMDD.sqlite3`
- **Zip 壓縮檔**: `cbdb_YYYYMMDD.zip`
- **Metadata 檔案**: `cbdb_YYYYMMDD.json`
- **歷史目錄**: `history/cbdb_YYYYMM/`
- **Metadata 目錄**: `metadata/YYYY-MM/`

## 故障排查

### HuggingFace 認證失敗

**錯誤訊息**：
```
錯誤: 未找到 HuggingFace 認證
```

**解決方案**：
```bash
# 重新登入
hf auth login

# 驗證認證狀態
hf auth status

# 或使用 Token 環境變數
export HF_TOKEN=hf_你的token
```

### zip 命令未找到

**錯誤訊息**：
```
錯誤: 缺少必要工具：
  - zip (請安裝: sudo apt-get install zip)
```

**解決方案**：
```bash
sudo apt-get update
sudo apt-get install zip
```

### hf 命令未找到

**錯誤訊息**：
```
錯誤: 缺少必要工具：
  - hf (請安裝: pipx install huggingface-hub)
```

**解決方案**：
```bash
# 安裝 pipx
sudo apt-get install -y pipx

# 安裝 huggingface-hub
pipx install huggingface-hub

# 確保 PATH 包含 ~/.local/bin
pipx ensurepath

# 重新載入 shell 或重新登入
source ~/.bashrc
```

### 匯出失敗

**錯誤訊息**：
```
找不到匯出檔案 db-data/cbdb_20241223.sqlite3
```

**解決方案**：
1. 檢查 `export-daily-sqlite.sh` 的執行日誌
2. 確認數據庫連接配置正確（`.env` 中的 `DB_CONNECTION` 和相關參數）
3. 檢查磁盤空間是否充足
4. 手動執行匯出腳本查看詳細錯誤：
   ```bash
   bash scripts/export-daily-sqlite.sh
   ```

### 上傳失敗

**錯誤訊息**：
```
Error: Cannot upload to cbdb/cbdb-sqlite: permission denied
```

**解決方案**：
1. 確認 HuggingFace Token 具備 Write 權限
2. 確認您是數據集的 collaborator
3. 聯繫數據集管理員添加權限

### 磁盤空間不足

**錯誤訊息**：
```
No space left on device
```

**解決方案**：
```bash
# 檢查磁盤使用情況
df -h

# 清理舊的 SQLite 匯出檔（如果未自動清理）
rm -f db-data/cbdb_*.sqlite3
rm -f db-data/cbdb_*.zip
rm -f db-data/cbdb_*.json

# 清理 Laravel 日誌（如果過大）
truncate -s 0 storage/logs/laravel.log
```

<Warning>
匯出和壓縮過程需要至少 3GB 可用磁盤空間。建議保持 5GB 以上可用空間。
</Warning>

## 最佳實踐

### 定時任務建議

- ✅ 選擇系統負載較低的時段（如凌晨）
- ✅ 週期性檢查 cron 日誌確保任務正常執行
- ✅ 設置日誌輪轉避免日誌文件過大
- ✅ 在部署更新前手動執行測試

### 安全考量

- ✅ HuggingFace Token 使用 Fine-grained 類型，最小權限原則
- ✅ 定期輪換 Token（建議每 6 個月）
- ✅ 限制 Token 的存取範圍（僅 `cbdb/cbdb-sqlite` 數據集）
- ✅ 使用 `hf auth login` 而非環境變數存儲 Token

### 監控與告警

建議設置以下監控：

```bash
#!/bin/bash
# 監控腳本：檢查上次同步是否成功

LOG_FILE="/var/log/cbdb-sqlite-sync.log"
LAST_SUCCESS=$(grep -n "同步完成" "$LOG_FILE" | tail -1 | cut -d: -f1)
LAST_ERROR=$(grep -n "錯誤" "$LOG_FILE" | tail -1 | cut -d: -f1)

if [ "$LAST_ERROR" -gt "$LAST_SUCCESS" ]; then
    echo "警告：最近一次同步失敗"
    # 發送告警郵件或通知
fi
```
