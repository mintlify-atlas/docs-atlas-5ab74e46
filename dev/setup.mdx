## 前置需求

在開始之前，請確保你的開發環境已安裝以下工具：

    生產環境使用 MariaDB 10.3.39，開發環境建議使用相容版本。
  </Card>
</CardGroup>

### 安裝 Node.js 22（Ubuntu）

本專案要求 **Node.js 22** 和 **npm 10**。推薦使用 **nvm (Node Version Manager)** 來安裝和管理 Node.js 版本：

<Steps>
  <Step title="安裝 nvm">
    ```bash
    curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
    ```
  </Step>
  
  <Step title="載入 nvm">
    ```bash
    source ~/.bashrc
    # 如果使用 zsh，則執行：source ~/.zshrc
    ```
  </Step>
  
  <Step title="安裝 Node.js 22">
    ```bash
    nvm install 22
    nvm use 22
    ```
  </Step>
  
  <Step title="驗證版本">
    ```bash
    node --version  # 應該顯示 v22.x.x
    npm --version   # 應該顯示 10.x.x
    ```
  </Step>
</Steps>

## 克隆倉庫

從 GitHub 克隆專案到本地：

```bash
git clone https://github.com/cbdb-project/cbdb-online-main-server.git
cd cbdb-online-main-server
```

## 安裝依賴

### 安裝 PHP 依賴

使用 Composer 安裝 Laravel 和所有後端套件：

```bash
composer install
```

這將安裝以下主要套件：
- **Laravel 12.0** - 核心框架
- **Laravel Sanctum** - API 認證
- **Laravel UI** - 用戶界面腳手架
- **PHP-CS-Fixer** - 代碼格式化工具
- **PHPUnit 11.x** - 測試框架

### 安裝前端依賴

使用 npm 安裝前端構建工具和 UI 庫：

```bash
npm install
```

這將安裝：
- **Vite** - 前端構建系統（已從 Laravel Mix 遷移）
- **Vue 3** - 前端框架
- **AdminLTE 3** - 基於 Bootstrap 4 的後台管理模板
- **jQuery 3.5** - JavaScript 庫
- **DataTables** - 表格插件
- **Select2** - 下拉選擇增強插件

<Note>
如果遇到 `npm install` 錯誤，請參考 [前端依賴問題排查](/dev/frontend#前端依賴問題排查) 章節。
</Note>

## 配置環境變量

### 創建 .env 文件

複製範例環境配置文件：

```bash
cp .env.example .env
```

### 生成應用密鑰

```bash
php artisan key:generate
```

### 配置數據庫連接

編輯 `.env` 文件，設置數據庫連接參數：

```env .env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=cbdb_online
DB_USERNAME=cbdb_user
DB_PASSWORD=your_password

# 數據庫時區 - 必須與 APP_TIMEZONE 匹配
# 重要：使用數字偏移格式（如 +08:00），不要使用命名時區
DB_TIMEZONE=+08:00

# Migration 專用帳號（具備 schema 變更權限）
DB_MIGRATE_USERNAME=cbdb_migrate
DB_MIGRATE_PASSWORD=your_migrate_password
```

<Warning>
**數據庫時區配置重要提醒**

`.env` 中的 `DB_TIMEZONE` **必須**與 `config/app.php` 的 `timezone` 匹配！

- ✅ 正確設定：`DB_TIMEZONE=+08:00`（對應 `APP_TIMEZONE=Asia/Shanghai`）
- ❌ 錯誤設定：`DB_TIMEZONE=Asia/Shanghai`（會導致 "Unknown or incorrect time zone" 錯誤）

**原因**：MySQL 命名時區需要載入 time zone tables，否則會導致連接錯誤。如果不匹配，MySQL TIMESTAMP 欄位會產生 8 小時時區偏移。
</Warning>

### 帳號權限分離（重要）

為了降低事故風險，**務必**將一般應用連線與 migration 連線分離：

<AccordionGroup>
  <Accordion title="為什麼需要權限分離？">
    基線 migration 會觸及完整 schema，如果日常帳號擁有過高權限（DROP/ALTER/CREATE），可能在操作失誤時造成數據損失。
  </Accordion>
  
  <Accordion title="配置原則">
    - **應用帳號**（如 `cbdb_user`）：僅需 CRUD 權限（SELECT、INSERT、UPDATE、DELETE）
    - **Migration 帳號**（如 `cbdb_migrate`）：具備 schema 變更所需權限（CREATE、ALTER、DROP、INDEX）
  </Accordion>
</AccordionGroup>

## 運行數據庫遷移

### 創建數據庫

如果數據庫尚不存在，請先在 MySQL/MariaDB 中創建：

```sql
CREATE DATABASE cbdb_online CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 執行 Migration

運行所有數據庫遷移文件來創建表結構：

```bash
php artisan migrate
```

這將執行：
1. **基線 Schema** - `2025_01_01_000000_import_cbdb_schema.php` 導入完整的 CBDB 數據庫結構
2. **增量修改** - 所有後續的 migration 文件
3. **內部輔助表** - 如 `CBDB__NAME_FTS`（姓名搜索索引）、`CBDB__TRAD_SIMP_MAP`（繁簡映射）

<Note>
所有 Migration 都經過 **MySQL/MariaDB 和 SQLite 兼容性處理**，可在測試環境（SQLite）和生產環境（MariaDB）中使用相同的 migration 文件。
</Note>

### 導入繁簡映射數據

執行 Artisan 命令導入繁簡字符映射：

```bash
php artisan cbdb:import-trad-simp-map --truncate
```

## 編譯前端資源

### 開發環境

啟動 Vite 開發服務器（支持熱重載）：

```bash
npm run dev
```

開發服務器將運行在 `http://localhost:5173`，所有前端資源變更會自動重新編譯。

### 生產環境構建

編譯並優化前端資源：

```bash
npm run build
# 或
npm run prod
```

這將生成優化後的資源文件到 `public/build/` 目錄。

<Warning>
**重要提醒**

代碼提交前必須執行 `npm run build`，確保編譯後的文件與源代碼同步。否則部署後前端會顯示舊版本。
</Warning>

## 啟動開發服務器

### 使用 Artisan 內建服務器

```bash
php artisan serve
```

應用將運行在 `http://localhost:8000`。

### 同時運行前端開發服務器

在另一個終端窗口中運行：

```bash
npm run dev
```

這樣可以同時享受：
- PHP 代碼的即時更新（通過 `php artisan serve`）
- 前端代碼的熱重載（通過 Vite）

## 創建管理員帳號

使用 Artisan 命令創建第一個管理員用戶：

```bash
php artisan cbdb:manage-user
```

按照提示輸入：
1. **用戶名**（c_name）
2. **Email**
3. **密碼**
4. **角色** - 選擇 `super-admin`

<Tip>
更多用戶管理操作，請參考 [用戶管理指南](https://github.com/cbdb-project/cbdb-online-main-server/tree/main/.claude/skills/user-management.md)。
</Tip>

## 運行測試

驗證環境配置正確：

```bash
./vendor/bin/phpunit
```

所有測試應該通過（綠燈）。如果測試失敗，請檢查：
- 數據庫連接配置是否正確
- Migration 是否成功執行
- PHP 和 Node.js 版本是否符合要求

## 常見問題

<AccordionGroup>
  <Accordion title="npm install 失敗怎麼辦？">
    如果遇到依賴安裝錯誤：
    
    ```bash
    rm -rf node_modules
    npm cache clear --force
    npm install
    ```
    
    詳細排查步驟請參考 [前端依賴問題排查](/dev/frontend#前端依賴問題排查)。
  </Accordion>
  
  <Accordion title="Migration 執行失敗？">
    常見原因：
    1. **數據庫帳號權限不足** - 確保 Migration 帳號有 CREATE/ALTER/DROP 權限
    2. **字符集問題** - 確保數據庫使用 `utf8mb4` 字符集
    3. **時區配置錯誤** - 檢查 `DB_TIMEZONE` 是否使用數字偏移格式（如 `+08:00`）
  </Accordion>
  
  <Accordion title="前端資源 404 錯誤？">
    如果頁面無法加載 CSS/JS：
    1. 確認已執行 `npm run build`
    2. 檢查 `public/build/` 目錄是否存在編譯文件
    3. 清除瀏覽器緩存
  </Accordion>
</AccordionGroup>
