## Frontend Build System

### Vite Configuration

**Entry Points:**
- `resources/js/app.js` - Main UI components (AdminLTE, Bootstrap, Select2)
- `resources/js/datatables.js` - DataTables functionality
- `resources/js/jquery-global.js` - Exposes jQuery to global scope

**Build Commands:**

```bash
# Development mode (with hot reload)
npm run dev

# Production build (minified, optimized)
npm run build
# or
npm run prod
```

<Tip>
  Always run `npm run build` before committing changes to frontend code to ensure compiled assets are up to date.
</Tip>

## Database Compatibility Requirements

<Warning>
  **Critical:** All database features must be compatible with both MySQL/MariaDB (production) and SQLite (testing).
</Warning>

### Allowed Features

✅ Standard SQL syntax  
✅ Laravel Schema Builder methods  
✅ B-Tree indexes  
✅ Foreign keys (with proper migration handling)  
✅ CHAR, VARCHAR, TEXT, INT, BIGINT, DECIMAL, DATE, DATETIME, TIMESTAMP  

### Prohibited Features

❌ MySQL/MariaDB-specific functions (e.g., `FULLTEXT` with ngram parser)  
❌ Database-specific plugins or extensions  
❌ Raw SQL without SQLite compatibility checks  
❌ Database-specific syntax in migrations (use `is_mysql()` / `is_sqlite()` helpers)  

### Migration Helper Functions

**Location:** `database/migrations/helpers.php`

```php
// Check database type in migrations
if (is_mysql()) {
    // MySQL/MariaDB specific code
}

if (is_sqlite()) {
    // SQLite specific code or skip unsupported features
}
```

<Note>
  See [Database Setup](/setup/database) for detailed migration guidelines.
</Note>

## Code Quality Tools

### PHP-CS-Fixer

**Configuration:** `.php-cs-fixer.dist.php`

**Key Rules:**
- PSR-12 standard with same-line braces
- Short array syntax `[]` (not `array()`)
- Alphabetical import ordering
- Trailing commas in multi-line arrays

**Usage:**
```bash
# Format all code
./vendor/bin/php-cs-fixer fix

# Preview changes without modifying
./vendor/bin/php-cs-fixer fix --dry-run --diff
```

### Testing

```bash
# Run all tests
./vendor/bin/phpunit

# Run specific test file
./vendor/bin/phpunit tests/Feature/CodesControllerTest.php

# Run with verbose output
./vendor/bin/phpunit --verbose
```

## Environment Configuration

### Required Environment Variables

**Database:**
```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=cbdb
DB_USERNAME=your_user
DB_PASSWORD=your_password
DB_TIMEZONE=+08:00  # MUST match APP_TIMEZONE
```

<Warning>
  **Critical:** `DB_TIMEZONE` must be `+08:00` (numeric offset) and match `APP_TIMEZONE=Asia/Shanghai`. Using named timezones in `DB_TIMEZONE` will cause "Unknown or incorrect time zone" errors.
</Warning>

**Application:**
```env
APP_ENV=production
APP_KEY=base64:...
APP_DEBUG=false
APP_TIMEZONE=Asia/Shanghai  # GMT+8
APP_URL=https://input.cbdb.fas.harvard.edu
```

### Separate Migration Credentials

<Tip>
  For security, use separate database accounts for application operations and migrations:
</Tip>

```env
# Regular application user (CRUD only)
DB_USERNAME=cbdb_app
DB_PASSWORD=***

# Migration user (schema changes)
DB_MIGRATE_USERNAME=cbdb_migrate
DB_MIGRATE_PASSWORD=***
```

## Development Environment Setup

<Steps>
  <Step title="Install PHP 8.2+ (recommended 8.4)">
    ```bash
    php --version  # Verify PHP 8.2 or higher
    ```
  </Step>
  
  <Step title="Install Composer dependencies">
    ```bash
    composer install
    ```
  </Step>
  
  <Step title="Install Node.js 22.x and npm 10.x">
    ```bash
    node --version  # Should show v22.x.x
    npm --version   # Should show 10.x.x
    ```
    
    Recommended: Use [nvm](https://github.com/nvm-sh/nvm) to manage Node versions.
  </Step>
  
  <Step title="Install frontend dependencies">
    ```bash
    npm install
    ```
  </Step>
  
  <Step title="Build frontend assets">
    ```bash
    npm run dev   # Development with hot reload
    npm run build # Production build
    ```
  </Step>
  
  <Step title="Configure environment">
    ```bash
    cp .env.example .env
    php artisan key:generate
    # Edit .env with your database credentials
    ```
  </Step>
  
  <Step title="Run migrations">
    ```bash
    php artisan migrate
    ```
  </Step>
</Steps>
